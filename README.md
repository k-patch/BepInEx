# BepInEx.Cpp2IL

This project features a newly improved **Cpp2IL-based BepInEx** build pipeline, designed to provide a consistent development environment and enhanced runtime stability.

## 🚀 Key Features & Updates

The following improvements have been made to the project structure and dependency management:

### 1. 🐳 Cross-Platform Docker Build Environment
We have established a comprehensive build pipeline based on **Docker Compose** to eliminate environment configuration complexity. You can now build and package artifacts (.zip) for all target platforms (Windows, Linux, macOS) in a clean environment with a single command, regardless of the .NET SDK or framework installed on your host OS.

**✅ How to Build:**
```bash
# Run the full platform build (Publish) using Docker Compose
docker compose up builder
```
- Upon successful completion, compressed `BepInEx-{Platform}-{Arch}-{Version}.zip` files will be generated in the `bin/dist/` directory.
- NuGet packages and Cake build tools are cached in Docker volumes for faster subsequent builds.

### 2. ⬆️ Support for Latest Unity IL2CPP Metadata (Cpp2IL)
Modern Unity games often feature updated metadata structures. To provide stable support, we have updated the `Samboy063.Cpp2IL.Core` dependency to the latest pre-release version (build `1446`) from the official NuGet feed.
- **Benefit:** Significantly improved compatibility for dumping and loading metadata from the latest Unity IL2CPP games, resolving previous parsing errors.

## 📂 Project Structure
- `Dockerfile` & `compose.yml`: .NET 6.0-based Docker configuration for multi-platform builds.
- `build.sh` & `build/Build.csproj`: Cake Frosting-based scripts for compiling and packaging the project.
- `Runtimes/Unity/BepInEx.Unity.IL2CPP/`: Core runtime code for IL2CPP environments, utilizing updated reverse engineering packages.

## 🛠️ IL2CPP Plugin Development Guide

When developing mods for IL2CPP-based games (e.g., *Supermarket Simulator*), you can use the latest template package (`ADDB.BepInEx.Templates`) to quickly scaffold your project.

### Creating a New Project

You can generate a plugin template using Docker without needing .NET installed locally. Run the following command in your terminal:

```bash
docker run --rm -v $(pwd):/app -w /app mcr.microsoft.com/dotnet/sdk:6.0 bash -c \
  "dotnet new --install ADDB.BepInEx.Templates && \
   dotnet new bepinexil2cppmod -n MyPlugin -G 'Supermarket Simulator' -A '\$(LocalAppData)Low\Nokta Games\Supermarket Simulator'"
```

**Parameter Details:**
*   `-n`: The name of your plugin project (e.g., `MyPlugin`).
*   `-G`: The name of the target game (e.g., `'Supermarket Simulator'`).
*   `-A`: (Windows) The game's AppData save location (usually `\$(LocalAppData)Low\Developer\Game`).

Once successful, a `MyPlugin` folder will be created with BepInEx configurations and build properties pre-defined for IL2CPP modding. You can then implement your mod by referencing the interop assemblies generated by BepInEx.
